# Week3 Activity 2

The ER diagram demonstrated a user centered design of a financial money exchange application.

There are 5 entities:
1. user - The person who uses the application. It contains information about the person.
2. security - The security approach applies to the user. One user may enable multiple login way such as text password, facial recognization etc.
3. account - The user may open multiple accounts in the application. Each account has its own balance in certain currency.
4. transaction - It manages all money exchange transactions of the user, recording all the information including source and destination accounts as well as the amount.
5. alarm - User may add alarms to allow applicaiton to notify the events when the exchange rate reaches the set threshold. A user may set multiple alarms for single or multiple currencies.
