# Week4 Activity 3

The two usecase diagrams demonstrated user/account use cases and money exchange transaction use cases separately.

* In the first diagram, both end user and the administrator need to create their own login information, including user profiles and credentials which is mandatory to secure the process. Moreover, end user can request opening an account and view his/her accounts information and status, while the adminstrator can manage end users, view his profiles and accounts, approve the openning request etc.
* In the second diagram, both end user and the administrator could view current currency exchange rate and history exchange transactions. In addition, end user can perform currency exchange, while the administrator can generate report of transactions based on various filters, such as date range, end user id or currency filters.
